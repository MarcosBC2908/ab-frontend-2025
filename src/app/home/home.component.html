2. Proyecto Full Stack

Hemos desarrollado una aplicación completa que incluye:

  • Frontend: Angular
  • Backend: Node.js + Express

REST (Representational State Transfer) es un estilo arquitectónico que define un conjunto de restricciones para crear servicios web escalables, utilizando los métodos estándar de HTTP: GET, POST, PUT, DELETE.

3. Frontend con Angular

  • Lenguaje: TypeScript
  • Tipos de datos: string, number, boolean, array, any, etc.
  • Comandos:
    • npm install: instala las dependencias del proyecto
    • ng serve: levanta el servidor de desarrollo
  • Crear componente: ng generate component nombre o ng g c nombre
  • NgModel: Directiva para binding bidireccional, requiere importar FormsModule
  • CommonModule: Incluye directivas básicas como *ngIf, *ngFor; se importa desde angular/common
  • Router: Define rutas con RouterModule. Permite navegación entre vistas.
  • Comunicación con backend: Usamos fetch() para hacer peticiones HTTP
  • Tipos de peticiones: GET, POST, PUT, DELETE
  • Archivos del componente:
    • .ts: lógica del componente
    • .html: plantilla
    • .css: estilos
    • .spec.ts: pruebas (opcional)
Documentación oficial de Angular

4. Backend con Node y Express

  • Lenguaje: JavaScript (Node.js)
  • Framework: Express.js
  • Comandos:
    • npm install: instala dependencias
    • npm start: inicia el servidor
  • Arquitectura:
    • index.js: punto de entrada
    • routes/: define rutas
    • controllers/: contiene la lógica de cada ruta
  • Postman: herramienta para probar endpoints. Permite enviar peticiones HTTP y ver respuestas.
  • CRUD: Operaciones básicas: Create, Read, Update, Delete
Documentación de Express | Sitio web de Postman

5. Control de versiones con Git

Todo el equipo ha trabajado en un único repositorio remoto usando ramas individuales.

  • Durante el proyecto se aplicaron principios de Scrum (metodología ágil): planificación, tareas, retrospectiva.
  • Comandos usados:
    • git clone: clona el repositorio remoto
    • git fetch: actualiza referencias desde el remoto
    • git pull: descarga y fusiona cambios remotos
    • git push: sube commits locales al repositorio remoto
    • git commit: guarda cambios con un mensaje
